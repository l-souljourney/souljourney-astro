## 使用原则

- 新的活跃技术文档继续写在 `docs/`
- 历史材料优先归档，不直接删除
- 对接开发默认只认活跃文档；同主题若存在历史草案，以活跃文档为准
- 根级持续更新记录只写 `update.md`
