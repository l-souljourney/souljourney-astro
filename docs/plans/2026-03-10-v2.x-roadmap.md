## v2.1 — Engine v3.8 对接适配

**目标：** Astro 博客能接收 Engine 自动推送的内容，自动构建上线
**依赖：** Engine v3.8 开发完成，GitHub Contents API 推送能力就绪
**预估工作量：** 1-2 天

### 开发清单

| # | 任务 | 说明 |
|---|------|------|
| D1 | 扩展 categories 值域 | `content.config.ts` 新增 Engine 使用的分类：`investment-notes`、`market-review`、`weekly-report` 等 |
| D2 | 新增 `source` frontmatter 字段 | 可选字段，值为 `engine` / `manual`，区分自动发布和手动发布 |
| D3 | 建立 commit 规范文档 | `content(zh):` / `content(en):` scope + committer identity 规范 |
| D4 | 端到端验证 | Engine API → GitHub Contents → Actions build → 双线部署，全链路走通 |
| D5 | 构建失败通知 | Actions 失败时通知（GitHub notification 或 webhook） |

#### Engine 侧契约（不在 Astro 仓库内，但需确认）

| # | 确认项 | 说明 |
|---|--------|------|
| E1 | Frontmatter 字段完全匹配 | Engine 输出的 .md 严格遵循 `content.config.ts` schema |
| E2 | GitHub Contents API 推送 | Engine 使用 REST API 而非 git push，从国内 FC 稳定可用 |
| E3 | Committer identity | `souljourney-engine <engine@l-souljourney.cn>` |
| E4 | Commit message 格式 | `content(zh): {article-title}` |

#### 验收标准

- [ ] Engine 推送一篇测试文章 → GitHub 收到 commit
- [ ] Actions 自动触发构建 → 双线部署成功
- [ ] 新文章在国内外均可访问
- [ ] `git log --author="souljourney-engine"` 可筛选 Engine 发布的内容

## v2.3 — AI 增强展示（对接 Engine v3.9）

**目标：** 在文章页面展示 Engine AI 分析结果
**依赖：** Engine v3.9 上线（AI cognition_gap + historical_mirrors 可用）
**预估工作量：** 2-3 天

### 开发清单

| # | 任务 | 说明 |
|---|------|------|
| G1 | AI 摘要 + 要点展示 | 文章页新增 `ai_essence.summary` + `key_points` 渲染区块 |
| G2 | 认知差异展示 | 展示 `cognition_gap`（人类观点 vs AI 修正），当 `has_discrepancy: true` 时高亮 |
| G3 | 历史相似文章 | 展示 `historical_mirrors` 关联文章列表，含相似度百分比 |
| G4 | 质量评分展示 | 文章卡片或详情页展示 `score` 评分（可选） |
| G5 | 字段缺失兼容 | `ai_essence` 为空或部分字段缺失时优雅降级，不报错不留白 |

#### 数据来源说明

v2.3 阶段 AI 数据有两种来源：
1. **Astro 博客**：Engine 生成的 .md frontmatter 中可新增 `ai_essence` YAML 字段（需扩展 schema）
2. **小程序**：通过 OSS JSON 的 `ai_essence` 对象获取（已有定义）

Astro 博客选择方案 1 — 在 markdown frontmatter 中嵌入 AI 数据，构建时静态渲染。

#### 验收标准

- [ ] 有 AI 数据的文章展示摘要、要点、认知差异
- [ ] 无 AI 数据的文章（历史文章）正常展示，无报错
- [ ] 历史相似文章可点击跳转

## 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| GitHub API 从国内 FC 不稳定 | Engine 无法推送内容 | 3 次重试 + OSS 暂存降级 |
| v2.2 页面去重引入回归 | 中英文站点功能异常 | 逐页面重构，每页提交前 E2E 验证 |
| COS 全球加速费用 | 月度成本增加 | 博客流量小，预计 < 1 元/月 |
| Cloudflare Pages 构建与 Actions 构建不一致 | 国内外内容不同步 | 同一 commit 触发，内容源一致；如需完全一致则统一用 Actions 构建 |
| Engine categories 与 Astro schema 不匹配 | 构建失败 | v2.1 中严格对齐 schema，Engine 侧做校验 |
