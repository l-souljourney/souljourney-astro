## 2. OpenSpec 执行策略（个人开发最佳实践）

每个版本使用一个独立 change，遵循：

1. `openspec list --json` 检查当前 active change
2. 每次只保留一个 in-progress change
3. 单 change 控制在 `<= 12` tasks
4. 每个 task 2 小时内完成，并有可验证验收
5. 完成后立刻 archive，再启动下个版本 change

建议命令节奏：

```bash
# 新版本开始
/opsx:new <change-name>
/opsx:ff

# 实施（按 task 执行，不做一次性大改）
/opsx:apply <change-name>

# 完成后归档
/opsx:archive <change-name>
```
