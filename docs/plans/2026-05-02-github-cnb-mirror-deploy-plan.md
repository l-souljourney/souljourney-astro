# GitHub -> CNB Mirror -> COS 切换记录

**日期：** 2026-05-02  
**状态：** completed  
**性质：** 历史切换摘要，不再作为当前公开主入口文档

---

## 1. 这份文档记录什么

这份文档记录 `souljourney-blog` 在 `2026-05-02` 完成的一次发布链路切换：

- 保留 GitHub 作为唯一代码源
- 将腾讯云侧构建与部署执行收敛到 CNB
- 不再由 GitHub 直接承担国内主站发布

它的作用是保留历史背景，而不是继续充当当前公开运维手册。

---

## 2. 切换结果

本轮切换完成后，公开可理解的链路口径变为：

`GitHub main 更新 -> GitHub 构建与发布健康检查 -> 同步到 CNB mirror -> CNB 侧构建并完成国内公开发布`

同时：

- GitHub 继续承担唯一代码源角色
- Cloudflare Pages 继续保留 GitHub 集成部署面
- 国内主站发布不再由 GitHub 直接写入

---

## 3. 为什么做这次切换

这次切换的核心动机是：

- 降低 GitHub-hosted runner 直连国内对象存储时的长尾发布问题
- 将“代码源”和“国内部署执行器”职责拆开
- 让当前发布链路更稳定、更容易继续演进
