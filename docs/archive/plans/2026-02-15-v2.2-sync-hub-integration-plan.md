# v2.2 GitHub Sync Hub Integration Implementation Plan

**Goal:** Implement automated synchronization of repository events to `souljourney-code` via GitHub Actions.

**Architecture:** A new GitHub Workflow will listen for `issues` and `pull_request` events and forward the payload to the Hub repository using `repository_dispatch`.

**Tech Stack:** GitHub Actions, cURL, GitHub REST API.

---

### Task 1: Create Dispatch Workflow

**Files:**
- Create: `.github/workflows/dispatch-to-hub.yml`

**Step 1: Write the workflow content**

```yaml
name: Dispatch to Sync Hub

on:
  # Listen to all issue changes
  issues:
    types: [opened, edited, deleted, closed, reopened, labeled, unlabeled, assigned, unassigned, milestoned, demilestoned]
  # Listen to PR closures (including merges)
  pull_request:
    types: [closed]

jobs:
  dispatch:
    runs-on: ubuntu-latest
    steps:
      - name: Send Event to Hub
        run: |
          curl -L \
            -X POST \
            -H "Accept: application/vnd.github+json" \
            -H "Authorization: Bearer ${{ secrets.GH_SYNC_TOKEN }}" \
            -H "X-GitHub-Api-Version: 2022-11-28" \
            https://api.github.com/repos/l-souljourney/souljourney-code/dispatches \
            -d "{\"event_type\": \"github_sync\", \"client_payload\": {\"repo\": \"${{ github.repository }}\", \"event\": ${{ toJson(github.event) }}}"
```

**Step 2: Verify file creation**

Run: `ls -l .github/workflows/dispatch-to-hub.yml`
Expected: File exists.

**Step 3: Commit**

```bash
git add .github/workflows/dispatch-to-hub.yml
git commit -m "feat: add GitHub Sync Hub dispatch workflow"
```
